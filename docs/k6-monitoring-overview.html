InfluxDB 時間戳碰撞問題 & 解法

✘ 問題 (Before)

// VU1 和 VU2 在同一毫秒內呼叫： VU1: counter.add(1) --> InfluxDB point: {measurement: "ws_connected", value: 1, time: T} VU2: counter.add(1) --> InfluxDB point: {measurement: "ws_connected", value: 1, time: T} SAME timestamp + tagset = UPSERT (覆寫!)
⚠ Grafana 顯示: ws_connected = 1 (預期為 2)

✔ 解法 (After)

// 加入唯一 tag 使 tagset 不同： VU1: counter.add(1, {room: "A", student: "1"}) --> InfluxDB point: {ws_connected, value: 1, room=A, student=1, time: T} VU2: counter.add(1, {room: "A", student: "2"}) --> InfluxDB point: {ws_connected, value: 1, room=A, student=2, time: T} DIFFERENT tagset = 獨立的資料點!
✅ Grafana 顯示: SUM(ws_connected) = 2
關鍵概念：InfluxDB 以 measurement + tagset + timestamp 作為 unique key。 相同的 key 會觸發 UPSERT (覆寫)。 只需在 .add() 時加入唯一 tag (如 room + student) 即可避免碰撞。
