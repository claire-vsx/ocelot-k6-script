04 — Quick Start ● 4 步驟跑起來 P.04 / 15
01

安裝依賴

先裝 K6 與 Node 套件

brew install k6 pnpm install
02

設定環境變數

填入 token、org id、URL

cp .env.example \ .env.local vim .env.local
03

編譯 TS

vite 把 .ts 編成 dist/*.js

pnpm build # 或監聽變更 pnpm build:watch
04

執行測試

載入 env 後執行，輸出到 InfluxDB

source .env.dev pnpm test:multi-room:influxdb # 不輸出指標的基本版 pnpm test:multi-room
Tip · 想看資料？ 加 :influxdb 後綴 → 自動輸出到 InfluxDB e.g. pnpm test:multi-room:influxdb

k6 run dist/load-test.js

11 — Metrics ● 關鍵指標體系 P.11 / 15
Rate
「N 次中成功幾次」— 用來訂 SLI / SLO 的成功率指標。
  • student_connected> 90%
  • student_seated> 90%
  • seat_within_3s> 95%
  • answers_submitted> 80%
  • teacher_connected> 90%
  • events_received> 90%
Trend
時間分布 — 看 p95 / p99 表現。
  • seat_time< 3000ms
  • submit_time< 3000ms
  • ws_connect_time< 500ms
  • quiz_received_time—
  • delivery_quiz_created→ Students
  • delivery_student_submitted→ Teacher
Counter
事件計數 — 看「發生過幾次」。
  • ws_event_quiz_createdevent
  • ws_event_student_submittedevent
  • ws_unexpected_close⚠ alert
  • http_error_500 / 502 / 503⚠ alert
  • http_success_choose_seat✓
  • room_lesson_createdsetup
所有 metric 都帶 { room, student } 標籤 避免 InfluxDB 時間戳碰撞 P.11
